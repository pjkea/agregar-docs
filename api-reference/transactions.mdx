## List transactions

```bash
GET /api/v1/transactions
```

Returns a paginated list of transactions for your institution.

### Query parameters

| Parameter | Type | Description |
|---|---|---|
| `page` | integer | Page number (default: 1) |
| `page_size` | integer | Results per page (default: 50, max: 200) |
| `start_date` | ISO 8601 | Filter: created at or after |
| `end_date` | ISO 8601 | Filter: created before |
| `decision` | string | `ALLOW`, `FLAG`, `STEP_UP`, `HOLD`, `BLOCK` |
| `customer_id` | UUID | Filter by customer |
| `channel` | string | `MOBILE`, `ATM`, `POS`, `WEB`, `USSD` |
| `min_amount` | number | Minimum transaction amount |
| `max_amount` | number | Maximum transaction amount |
| `min_score` | integer | Minimum combined risk score |

### Example

```bash
curl "https://monitor.agregartech.com/api/v1/transactions?decision=FLAG&min_score=90&page_size=25" \
  -H "X-API-Key: your_api_key_here"
```

## Get transaction detail

```bash
GET /api/v1/transactions/{transaction_id}
```

Returns full detail for a single transaction including all scoring layers, triggered rules, and ML explanation.


## Export transactions

```bash
GET /api/v1/transactions/export
```

Downloads a CSV of transactions matching the same query parameters as the list endpoint.

```bash
curl "https://monitor.agregartech.com/api/v1/transactions/export?start_date=2026-06-01&decision=FLAG" \
  -H "X-API-Key: your_api_key_here" \
  -o transactions.csv
```

<Note>
  Exports are capped at 10,000 rows. For larger datasets, page through the list endpoint or contact Agregar for a bulk extract.
</Note>
